# Fork-инфраструктура

Этот каталог — **дом для всех изменений нашего форка**. Upstream (`stablyai/orca`)
никогда не создаёт файлы здесь, поэтому всё в `fork/` имеет **нулевую вероятность
merge-конфликта**. Любую кастомизацию по умолчанию делаем здесь, а в upstream-файлы
вносим минимум точек подключения.

## Почему так

Upstream выпускает **~60 коммитов в день** (≈1800/мес). Боль форка — не git-команды,
а **площадь пересечения** наших правок с часто меняющимися upstream-файлами. Стратегия
сводится к одному: держать эту площадь минимальной.

## Три правила

1. **Новый код → новый файл** (в `fork/` или рядом, с уникальным именем). Новый файл
   не конфликтует никогда.
2. **Правку upstream-файла оборачиваем сентинелом** `// [FORK]` … `// [/FORK]`
   (или `/* [FORK] */` для CSS). Тогда после конфликта вставку видно мгновенно, а
   `fork/scripts/divergence.sh` показывает всю площадь расхождения.
3. **Универсальное улучшение → PR в upstream**, не в форк. Если правка не про
   брендинг и не проприетарна — её дешевле отдать наверх, чем тащить вечно.

## Git-workflow

Merge-based, **никогда не rebase** (rebase требует force-push — яд для команды).

```bash
git fetch upstream
git merge upstream/main        # делать ЧАСТО — 2–3 раза в неделю, не раз в месяц
# разрешить конфликты (их площадь видна через divergence.sh)
pnpm install && pnpm typecheck && pnpm lint && pnpm test
```

Чем чаще мерджим — тем мельче конфликты. Накопить 1800 коммитов за месяц = ад;
смерджить 150 за 2–3 дня = рутина.

CI-страховка: `.github/workflows/upstream-sync.yml` пытается мерджить upstream
по расписанию, гоняет проверки и открывает PR (или issue при конфликте).

## Карта «швов» (риск правки upstream-файла)

🟢 **Низкий риск** (предпочитать):
- Тема/цвета/радиусы/шрифты → `fork/styles/fork-overrides.css` (НЕ трогать `assets/main.css`).
- Новая Zustand-слайс → новый файл `fork/store/*.ts` + 1 импорт/spread в `store/index.ts`.
- Переопределение shadcn-примитива → `src/renderer/src/components/ui/*` (меняется редко).

🟡 **Средний риск** (можно, но под `[FORK]`):
- Новый раздел/вид: новый lazy-page + ветка в `App.tsx` + кнопка в `SidebarNav.tsx` + тип в `store/slices/ui.ts`.
- Новый флаг в `GlobalSettings` (`src/shared/types.ts`).

🔴 **Высокий риск** (избегать переписываний, только точечные вставки):
- `src/renderer/src/App.tsx` (~2700 строк, меняется постоянно).
- `store/slices/ui.ts`, `tabs.ts`, `terminals.ts`.
- `components/native-chat/*` — здесь живёт «Cursor-like» чат, но это самая активная зона upstream.

## Подводные камни (иначе CI красный после мерджа)

- **Локализация**: новые UI-строки идут через i18n + `pnpm sync:localization-catalog`
  (линт проверяет каталог на 5 языках).
- **`max-lines` отключать нельзя** — расширяемся новыми файлами.
- **STYLEGUIDE.md** обязателен для UI — только токены, без магических цветов.
